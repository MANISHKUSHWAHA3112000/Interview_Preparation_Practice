## 🔹 Technical Answers

**Angular**
1. **Angular 17 vs earlier versions, migration to 20**  
   *Answer:*  
   “Angular 17 introduced a new control flow syntax (`@if`, `@for`) which improves readability and performance compared to `*ngIf` and `*ngFor`. It also enhanced hydration for server-side rendering and improved build performance. When migrating to Angular 20, I would first upgrade dependencies, run compatibility checks, and use Angular’s schematics to handle breaking changes. I’d also set up automated tests to ensure no regressions during migration.”

2. **Change detection optimization**  
   *Answer:*  
   “Angular uses the Zone.js mechanism for change detection. By default, it checks all components, but we can optimize using `ChangeDetectionStrategy.OnPush` and immutability. For large apps, I rely on `trackBy` in `*ngFor`, lazy loading modules, and RxJS operators to reduce unnecessary re-renders.”

3. **Lazy loading importance**  
   *Answer:*  
   “Lazy loading allows modules to be loaded only when needed, reducing initial bundle size and improving performance. For example, in a large enterprise app, the admin module can be lazy-loaded so regular users don’t download unnecessary code.”

4. **Security in Angular apps**  
   *Answer:*  
   “To prevent XSS, Angular’s built-in sanitization automatically escapes HTML. For CSRF, I use tokens validated by the backend. I also enforce Content Security Policy headers and avoid using `innerHTML` directly.”

---

**Java / Backend**
1. **Java 17 features**  
   *Answer:*  
   “Java 17 introduced sealed classes, records, and pattern matching for `switch`. Sealed classes restrict inheritance, improving design safety. Records reduce boilerplate for data classes. These features make code more concise and maintainable.”

2. **REST API design**  
   *Answer:*  
   “I follow REST best practices: use nouns for endpoints (`/users`), proper HTTP methods (GET, POST, PUT, DELETE), versioning (`/api/v1`), and standardized error responses with meaningful status codes. I also document APIs with OpenAPI/Swagger.”

3. **Quarkus vs Spring Boot**  
   *Answer:*  
   “Spring Boot is mature and widely adopted, great for enterprise apps. Quarkus is optimized for cloud-native and GraalVM, offering faster startup and lower memory footprint. I’d choose Quarkus for microservices in Kubernetes, and Spring Boot for complex enterprise systems with rich ecosystem support.”

4. **Hibernate lazy vs eager loading**  
   *Answer:*  
   “Lazy loading fetches data only when accessed, reducing initial load. Eager loading fetches everything upfront, which can cause performance issues if not needed. I prefer lazy loading with proper `JOIN FETCH` queries when necessary.”

## 🔹 Coding Questions (Sample Answers)

**Java – First Non-Repeating Character**
```java
import java.util.*;
import java.util.stream.*;

public class FirstUniqueChar {
    public static void main(String[] args) {
        String str = "swiss";
        Character result = str.chars()
            .mapToObj(c -> (char)c)
            .collect(Collectors.groupingBy(c -> c, LinkedHashMap::new, Collectors.counting()))
            .entrySet().stream()
            .filter(e -> e.getValue() == 1)
            .map(Map.Entry::getKey)
            .findFirst()
            .orElse(null);
        System.out.println(result); // Output: w
    }
}
```

**Angular – Service with RxJS**
```typescript
@Injectable({
  providedIn: 'root'
})
export class EmployeeService {
  constructor(private http: HttpClient) {}

  getEmployees(): Observable<Employee[]> {
    return this.http.get<Employee[]>('/api/employees').pipe(
      retry(2),
      catchError(error => {
        console.error('Error fetching employees', error);
        return throwError(() => new Error('Failed to load employees'));
      })
    );
  }
}
```
